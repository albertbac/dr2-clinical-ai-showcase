# Public Disclosure Policy

## Principle

Minimal exposure. Strong documentation. Zero avoidable leakage.

## What this repository is allowed to publish

- reconstructed public descriptions
- high-level architecture narratives
- high-level workflows
- public-safe positioning text
- screenshot shotlists for manual capture
- synthetic-data guidance when safe material can be created later

## What this repository does not publish

- production source code
- live datasets
- raw operational records
- internal infrastructure details
- private access material
- internal prompts or private playbooks
- unredacted payloads or logs
- customer-specific or institution-specific content

## Publication rule

If a statement is not clearly safe, it should not be published.
If confidence is not high enough, the file must state the gap instead of inventing detail.
