# Public-safe Guidelines

## Core rule

Describe systems. Do not expose systems.

## Safe inputs for this repository

- product purpose
- safe UI labels
- safe architectural summaries
- high-level workflows
- non-sensitive stack descriptions
- manually curated public screenshots captured later

## Unsafe inputs for this repository

- copied production code
- copied data
- copied internal prompts
- copied infrastructure notes
- copied deployment instructions
- copied raw logs or payloads

## Writing rule

Use sober technical language.
State uncertainty when needed.
Never fill gaps with invented detail.
