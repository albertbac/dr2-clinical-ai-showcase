# Template-driven Ishikawa Diagram Builder

## Summary
A structured Ishikawa builder that turns guided inputs into consistent SVG and PNG outputs.

## Primary user
Quality or process improvement team

## Secondary user
Operations or design facilitator

## Problem solved
Teams often create fishbone diagrams manually, with inconsistent layout and low reuse.

## Functional workflow
The user selects a template, fills guided fields, previews the result, and exports a diagram artifact in a stable visual format.

## Public-safe technical scope
Public-safe scope: template registry, guided input, preview, export, and isolation of rendering logic.

## High-level integrations
High-level only: identity, storage, and render services.

## What stays private and why
Template-zone definitions, internal assets, storage wiring, and project-host integration details remain private.
